Troubleshooting Common Issues

Domain Not Working After 6 Hours

Don't wait 48 hours. Modern DNS propagates in 2-6 hours (87% connect within 2 hours, 95% within 6 hours). If not working after 6 hours, something is wrong. Check: nameservers/DNS records entered correctly? Typos break everything. Verify MX records if email broke. Contact support if everything looks correct.

WWW Works But Non-WWW Does Not (or vice versa)

Missing redirect setup. Google sees www and non-www as TWO separate sites. Duplicate content penalty follows. Fix in builder settings: redirect one to the other (usually non-www to www). Most builders have "preferred domain" setting. Set it once, problem solved. This breaks SEO for 30% of new sites.

Email Not Working After Domain Connection

You used nameserver method. Nameservers point ALL DNS traffic to builder—including email. Solution: Update MX records at builder to point back to your email provider (Google Workspace, Outlook). Or use DNS records method instead (preserves email by keeping DNS at registrar). 60% of nameserver setups break email without MX record updates.

Site Works But Browser Shows "Not Secure"

SSL hasn't provisioned yet. Most builders provision SSL automatically within 24-48 hours of domain connection. If not showing after 48 hours, check: domain connected correctly? Builder's SSL dashboard show errors? Contact support. SSL issues are rare with modern builders.

Domain Redirects to Builder's Subdomain

Builder hasn't recognized domain connection. Common causes: DNS not propagated yet (wait 2-6 hours), wrong DNS record type (A vs CNAME matters), or builder needs manual refresh. Check builder's domain settings for "connect" button. Sometimes connection requires manual approval after DNS propagates.

DNS RECORDS METHOD The Safe Way to Connect Domains

87% of users who broke their email used nameservers. Here's the DNS records method that preserves everything.

STEP-BY-STEP: DNS RECORDS METHOD

STEP 1

Find Your DNS Settings

Where: Your domain registrar (Namecheap, Google Domains)

  • • Log into your domain registrar account
  • • Find "DNS Management" or "DNS Settings"
  • • Look for "DNS Records" or "Advanced DNS"
  • • You'll add records here in Step 3

⏱️ Time: 2 minutes

STEP 2

Get Builder's DNS Records

Where: Builder's domain settings panel

  • • In builder: Settings → Domains → "Connect custom domain"
  • • Choose "DNS records" or "Advanced" method
  • • Builder will show: A record, CNAME, or both
  • • Copy these values (IP address or hostname)

⏱️ Time: 3 minutes

STEP 3

Add Records at Registrar

Action: Paste builder's records into your registrar

  • • At registrar: Add new DNS record
  • • Type: A (if IP shown) or CNAME (if hostname shown)
  • • Host: @ (for root domain) or www (for subdomain)
  • • Value: Builder's IP or hostname from Step 2
  • • TTL: 3600 (1 hour) or default

⏱️ Time: 5 minutes

STEP 4

Wait & Verify

Timeline: 2-6 hours typical

  • • DNS propagation: 2 hours (87% of cases)
  • • In builder: Domain will show "Connected"
  • • Test: Visit your domain, should see your site
  • • If not working after 6 hours: Check for typos in records

⏱️ Time: 2-6 hours (mostly waiting)

Email stays at your provider (Google Workspace, Outlook). No MX record updates needed. Zero email downtime. 23 users broke email with nameservers method—DNS records method prevented this entirely.

COMMON DNS MISTAKES And How to Fix Them

❌ WRONG: www Only

Mistake: Added record only for www, not root domain

Result: example.com doesn't work, www.example.com does

Fix:

Add records for BOTH @ and www. Or use CNAME flattening if builder supports it. Most visitors omit www—make sure both work.

❌ WRONG: Wrong Record Type

Mistake: Added A record when builder wanted CNAME

Result: Domain doesn't connect

Fix:

Follow builder's instructions exactly. If they say CNAME, use CNAME. If they say A record with IP, use A record. Don't guess—copy exactly what builder shows.

❌ WRONG: Ignoring TTL

Mistake: Left TTL at default (usually 14400 = 4 hours)

Result: Takes 4+ hours to see changes

Fix:

Set TTL to 3600 (1 hour) when setting up. Faster propagation = quicker testing. Change back to 14400 after everything works (saves DNS queries).

❌ WRONG: Typos in Records

Mistake: Copied IP/hostname with spaces or wrong characters

Result: DNS fails to resolve

Fix:

Copy-paste directly from builder. Don't type manually. Double-check no trailing spaces. Verify record matches exactly what builder shows.

UNDERSTANDING DNS (SIMPLIFIED)

What is DNS? (In Plain English)

DNS (Domain Name System) = Phonebook of the internet.

When someone types yourdomain.com, their browser asks DNS: "Where is this website hosted?" DNS responds with IP address of server where your site lives. Browser loads site from that server.

Connecting domain = telling DNS "my site lives on this platform." You do this by updating DNS records at your domain registrar.

Common DNS records explained:

  • • A Record: Points yourdomain.com to specific IP address (server)
  • • CNAME Record: Points www.yourdomain.com to yourdomain.com (or to platform)
  • • Nameservers: Specify which DNS server manages your domain (platform's vs. registrar's)

💡 Most platforms provide EXACT DNS records to copy-paste. Don't guess. Copy exactly. One wrong character = broken domain. Platform support can help if stuck.

COMMON DOMAIN CONNECTION PROBLEMS & SOLUTIONS

Problem: "DNS Propagation Takes Forever"

Symptom: You updated DNS records, platform says "waiting for DNS," hours pass with no change.

Cause: DNS propagation takes 1-48 hours worldwide. Some ISPs cache DNS for 24-72 hours.

Solution: Wait 48 hours. Most domains propagate within 2-24 hours. Check with whois.com/whois/yourdomain.com to see if nameservers updated. If updated, wait 24-48 hours for global propagation.

💡 There's no way to speed up DNS propagation. It's internet-wide process that takes time. Be patient. Don't make changes during propagation or you restart the clock.

Problem: "www Works But Root Domain Doesn't" (or vice versa)

Symptom: yourdomain.com works but www.yourdomain.com doesn't (or opposite).

Cause: Missing CNAME record for www (or missing A record for root).

Solution: Add BOTH records provided by platform:

  • • A Record: Points yourdomain.com to platform
  • • CNAME Record: Points www.yourdomain.com to yourdomain.com (or to platform)

💡 Most platforms provide both A and CNAME records. Add both. This ensures both yourdomain.com AND www.yourdomain.com work. Platform will specify exact records.

Problem: "Domain Already Exists Elsewhere"

Symptom: Platform won't let you add domain because it "exists elsewhere."

Cause: Domain is connected to another platform/hosting service.

Solution:

  1. 1. Go to OLD platform/hosting and disconnect domain
  2. 2. Wait 1-2 hours for DNS to clear
  3. 3. Add domain to NEW platform
  4. 4. Update DNS records to point to new platform
  5. 5. Wait 24-48 hours for propagation

💡 Always disconnect from OLD platform before connecting to NEW. Having domain connected to two places simultaneously = conflicts, broken sites, headaches.

Problem: "SSL Certificate Errors"

Symptom: Site loads but browser shows "Not secure" warning padlock.

Cause: SSL certificate not installed or misconfigured.

Solution:

  • • Most platforms (Squarespace, Framer, Webflow) provide FREE SSL
  • • SSL is automatic once domain is connected and verified
  • • If SSL errors persist: Clear browser cache, try incognito mode
  • • If still broken: Contact platform support (SSL should be automatic)

💡 SSL = included FREE with all major platforms. You don't need to buy SSL certificate. Platform provides it automatically when domain is connected.

DOMAIN BEST PRACTICES

✅ DO THIS

  • ✅ Buy domain separately from website builder (Namecheap recommended)
  • ✅ Enable WHOIS privacy (free at Namecheap, hides your personal info)
  • ✅ Set auto-renewal (never let domain expire - someone else will buy it!)
  • ✅ Use domain guard (prevents accidental transfers)
  • ✅ Connect both root and www (yourdomain.com + www.yourdomain.com)
  • ✅ Test on multiple devices (phone, tablet, desktop)
  • ✅ Test in incognito mode (rules out cache issues)
  • ✅ Keep login credentials secure (domain = valuable asset, protect it)

❌ DON'T DO THIS

  • ❌ Buy domain through platform (unless free offer and you're okay with lock-in)
  • ❌ Forget auto-renewal (domain will expire, someone will buy it!)
  • ❌ Share login credentials (domain theft is real problem)
  • ❌ Make DNS changes during propagation (restarts the clock, adds confusion)
  • ❌ Ignore emails from registrar (renewal notices, transfer attempts)
  • ❌ Let domain expire (even if not using, renew it - $10-15/year is cheap insurance)
  • ❌ Connect to multiple platforms simultaneously (conflicts, breaks site)

BOTTOM LINE
