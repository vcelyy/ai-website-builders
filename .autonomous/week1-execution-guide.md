## Success Metrics

**Track these:**
- Daily visitor count (Google Analytics or Plausible)
- Engagement rate (comments, likes, shares)
- Traffic source (which platform performed best)
- Session duration (>2 minutes = quality traffic)

**Good performance:**
- Reddit: >100 visitors, 20+ upvotes
- Twitter: >50 link clicks, 20+ likes
- Indie Hackers: >200 visitors, 20+ upvotes
- LinkedIn: >50 visitors, 10+ likes
- Facebook: >30 visitors per group, 5+ reactions

**Great performance:**
- Double the good numbers
- Scale what works in Week 2-4

**Poor performance:**
- <50% of expected
- Adjust approach
- Try different angle/title
- Don't repeat same failing post

## Common Issues & Solutions

**Issue 1: Post gets downvoted (Reddit)**
- Solution: Don't delete. Learn from feedback.
- Next time: Try different title/angle

**Issue 2: Zero engagement (Twitter)**
- Solution: Thread didn't resonate. Try different hook.
- Next time: Test hooks mentioned in backup section

**Issue 3: Post removed (Facebook group)**
- Solution: You may have violated group rules
- Next time: Read rules carefully, engage first

**Issue 4: No comments (LinkedIn)**
- Solution: Normal for LinkedIn, lower engagement
- Next time: Try more provocative question at end

**Issue 5: Site crashes from traffic**
- Solution: GitHub Pages can handle this traffic
- If issue: Check GitHub Pages status, wait for recovery
