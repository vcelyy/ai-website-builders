Migration steps (the hard way)

Important: Squarespace doesn't allow content export. There's no "download my site" button. Migration is manual work. Plan for 4-6 hours, or pay a freelancer $200-400.

Step 1: Audit your Squarespace site (30 minutes)

List all pages. List all images (note filenames). List all blog posts. Map out your site structure. You'll need this to recreate everything.

Step 2: Download all assets (30-60 minutes)

Right-click and save every image. Or use Squarespace's built-in image manager to download in bulk. Save to organized folders: "hero images," "product photos," "blog images."

Step 3: Export all text content (1-2 hours)

Copy all text from each page into a Google Doc. Organize by page: "Home page," "About," "Services," etc. Include metadata (page titles, descriptions) for SEO. This is tedious but necessary.

Step 4: Choose and set up new platform (30 minutes)

Create account on Framer, 10Web, or Durable. Connect custom domain (or buy one if you were using Squarespace's default). Set up basic settings.

Step 5: Recreate site structure (1-2 hours)

Create all pages in new platform. Match Squarespace structure as closely as possible. Don't worry about exact design matching—it's impossible. Focus on content hierarchy and user flow.

Step 6: Copy content and rebuild (2-3 hours)

Paste text from Google Docs into new pages. Upload images. Recreate layouts manually. This takes the longest time. Use AI generation to speed up, then customize with your actual content.

Step 7: Set up redirects (30 minutes)

Keep Squarespace live for 30 days. Map old URLs to new URLs using 301 redirects. This preserves SEO. After 30 days, cancel Squarespace and remove redirects.

SEO preservation during migration

The #1 migration fear: Losing search rankings. Done right, migrations don't hurt SEO. Done wrong, you can lose months of progress.

Keep Squarespace live for 30 days

Don't cancel immediately. Build new site on subdomain (new.yoursite.com). Test everything. Only switch DNS when new site is perfect.

Match page URLs exactly

If Squarespace page is /services, make new page /services. Don't change /about-us to /about. URL changes hurt SEO.

Set up 301 redirects

Use redirect plugin or DNS-level redirects. Map every old URL to new URL. This tells Google "content moved permanently" and preserves rankings.

Keep meta titles and descriptions

Copy title tags and meta descriptions exactly. These are SEO signals. Changing them tells Google "this is different content."

THE MIGRATION CHALLENGE

🚫 Squarespace Export Limitations

What You CAN Export:

  • • XML export of basic content (pages, text)
  • • Images (must download individually)
  • • Blog posts (XML format)
  • • Basic structure (hierarchy, pages)

What You CANNOT Export:

  • • Direct import to AI builders (no 1-click migration)
  • • Squarespace-specific formatting (rebuild required)
  • • Image galleries (rebuild manually)
  • • Forms (rebuild, data lost)
  • • Product data (ecommerce requires manual export)

Reality: Squarespace doesn't want you to leave. Export is deliberately limited. Migration is manual copy-paste work. 5-10 hours for typical site (10-20 pages). 20+ hours for complex sites (ecommerce, 50+ pages).

🔗 SEO Preservation Risks

What Can Go Wrong:

  • • URL structure changes (loss of backlink value)
  • • Meta titles/descriptions not transferred (must recreate)
  • • Site structure changes (confuses Google)
  • • Downtime during migration (temporary ranking drop)
  • • Internal links break (404 errors hurt SEO)

How to Preserve SEO:

  • • Keep identical URL structure if possible
  • • Export all meta titles, descriptions (recreate manually)
  • • Set up 301 redirects for changed URLs
  • • Submit new sitemap to Google Search Console
  • • Monitor rankings post-migration (expect temporary dip)

STEP-BY-STEP MIGRATION GUIDE

📋 Phase 1: Preparation (1-2 hours)

Export from Squarespace:

  1. Export XML (Settings → Website → Export)
  2. Download all images (right-click, save each)
  3. Copy all text content (paste into Google Doc)
  4. Export products if ecommerce (CSV export)
  5. Document current URL structure

Choose AI Builder:

  • • Durable: For cost savings ($180/year flat)
  • • Framer: For better design (9.5/10 vs 7.5/10)
  • • 10Web: For WordPress power + lowest cost
  • • Mixo: For better AI content quality

🏗️ Phase 2: Build New Site (2-4 hours)

  1. Generate with AI: Enter your site description into chosen AI builder. Review generated site.
  2. Import Content: Copy-paste text from Squarespace export to new site page by page.
  3. Upload Images: Re-upload all downloaded images to new site (organize in folders).
  4. Rebuild Structure: Recreate page hierarchy, navigation menus, footer.
  5. Rebuild Forms: Recreate contact forms, signup forms. Export form data from Squarespace before disabling.
  6. Rebuild Ecommerce: Import product CSV to new platform. Set up payments (Stripe).
  7. Customize Design: Adjust colors, fonts, layout to match or improve on Squarespace site.

🔗 Phase 3: Technical Migration (1-2 hours)

  1. Set Up URL Structure: Create pages matching Squarespace URLs exactly (about → /about, etc.).
  2. Recreate Meta Data: Copy meta titles, descriptions from Squarespace to new site pages.
  3. Set Up Redirects: If URLs must change, set up 301 redirects (old URL → new URL).
  4. Test All Links: Click every internal link. Fix any 404 errors.
  5. Test Forms: Submit test forms. Confirm email delivery works.
  6. Test Ecommerce: Run test transaction. Confirm payment processing works.
  7. Mobile Test: Check site on phone. All pages must work on mobile.

🚀 Phase 4: Go Live (1 hour)

  1. Connect Domain: Point domain from Squarespace to new platform (DNS settings).
  2. Keep Squarespace Live: Don't cancel Squarespace yet. Keep for 2 weeks as backup.
  3. Monitor Traffic: Use Google Analytics to confirm traffic flows to new site.
  4. Check Rankings: Monitor Google rankings for 2 weeks. Expect temporary dip, then recovery.
  5. Cancel Squarespace: After 2 weeks of smooth operation, cancel Squarespace subscription.
